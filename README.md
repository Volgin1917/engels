# Engels System

Система «Engels» — вычислительное ядро для автоматизации историко-материалистического анализа. Преобразует неструктурированные исторические документы в верифицируемый граф причинно-следственных связей.

## ✨ Возможности

- **Извлечение сущностей и связей** — автоматическое распознавание исторических событий, персоналий, концепций и причинно-следственных связей между ними
- **Двухслойная верификация** — разделение на сырые данные (LLM/MCP гипотезы) и проверенные исследователем факты
- **Полная аудируемость** — каждая связь сопровождается цитатой из источника и идентификатором документа
- **Локальный инференс** — работа без внешних API по умолчанию (Ollama + CPU)
- **Экспорт в Obsidian** — интеграция с популярными инструментами исследователей
- **MCP-интеграция** — безопасное подключение внешних вычислительных узлов через DLP-анонимизацию

## 🛠 Технологический стек

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **Backend** | Strapi v4, Python, Celery | API, CMS, очередь задач |
| **Frontend** | React, TypeScript, @xyflow/react | Визуализация графов, верификация |
| **База данных** | PostgreSQL 15+ + pgvector | Граф сущностей, эмбеддинги, JSONB |
| **LLM** | Ollama (qwen2.5:7b, llama3:8b) | Локальный инференс на CPU |
| **Очередь** | Redis | Брокер сообщений для Celery |
| **Инфраструктура** | Docker Compose, Gitea Actions | Контейнеризация, CI/CD |

## 🚀 Быстрый старт

### Требования

- Docker и Docker Compose
- Минимум 8 ГБ ОЗУ (рекомендуется 16 ГБ для локального LLM)
- Python 3.10+ (для разработки)

### Установка

```bash
# Клонирование репозитория
git clone <repository-url>
cd engels-system

# Запуск всех сервисов
docker compose up -d

# Проверка статуса
docker compose ps
```

### Первоначальная настройка

```bash
# Инициализация базы данных
docker compose exec strapi npm run build
docker compose exec strapi npm run develop

# Загрузка тестовых данных (опционально)
make seed-data
```

Сервисы будут доступны по адресам:
- Strapi Admin: `http://localhost:1337/admin`
- React UI: `http://localhost:3000`
- PostgreSQL: `localhost:5432`
- Redis: `localhost:6379`

## 🔒 Безопасность

### Принципы безопасности

1. **Замкнутый контур** — локальный инференс по умолчанию, внешние API запрещены
2. **DLP-анонимизация** — перед отправкой по MCP имена, даты и топонимы заменяются маркерами
3. **Circuit Breaker** — автоматическое переключение на локальную обработку при сбоях внешних узлов
4. **Строгая маркировка** — все данные от внешних источников помечаются как `raw` и требуют верификации
5. **Без логирования чувствительных данных** — сырые тексты и токены никогда не записываются в логи

### Маршрутизация задач

| Тип задачи | Маршрут по умолчанию | Fallback |
|------------|---------------------|----------|
| NER, извлечение связей | Local Ollama | — |
| Тяжёлый анализ, кластеризация | MCP External | Local Ollama |
| Экспорт, генерация отчётов | Local Ollama | — |
