  - [x] PostgreSQL с расширением `pgvector`

  - [x] Ollama для локального инференса LLM
  - [x] Сети и тома для персистентности данных
- [x] **Инициализация БД**: Создание схемы в `database/init.sql`
  - [x] Таблицы: `sources`, `text_chunks`, `entities`, `relations`, `audit_log`
  - [x] HNSW-индексы для векторного поиска
  - [x] Триггеры для аудита изменений
- [x] **Конфигурация окружения**

  - [ ] `.env` для локальной разработки

  - [x] `make docker-up` — подъем инфраструктуры

  - [x] `make lint` / `make format` — линтинг и форматирование

## 🔹 Фаза 2: Граф знаний и Извлечение сущностей
**Срок:** Недели 4-5  
**Статус:** ⚪ Ожидание

### Цели
- Построение графа сущностей и связей из текста
- Реализация марксистской онтологии

### Задачи
- [ ] **Экстракция сущностей**
  - [ ] Промпты для LLM (Ollama) для выделения сущностей
  - [ ] Выделение типов: `класс`, `эпоха`, `тип_производства`, `персона`
- [ ] **Извлечение связей**
  - [ ] Определение отношений между сущностями
  - [ ] Типы связей: `принадлежит_к`, `действует_в`, `производит`, `эксплуатирует`
- [ ] **Построение графа**
  - [ ] Дедупликация сущностей (fuzzy matching + векторная близость)
  - [ ] Сохранение графа в БД
- [ ] **DLP и Анонимизация**
  - [ ] Детекция PII (Personal Identifiable Information)
  - [ ] Замена чувствительных данных перед отправкой в LLM
- [ ] **Верификация**
  - [ ] Флаг `needs_review` для сомнительных экстракций
  - [ ] Интерфейс для ручной проверки (подготовка к Фазе 3)

### Критерии готовности (DoD)
- ✅ Граф сущностей и связей строится автоматически из текста
- ✅ Сущности классифицированы согласно онтологии
- ✅ PII данные анонимизированы до обработки LLM
- ✅ Механизм дедупликации работает корректно
- ✅ Эксперт может отметить сущность как "требует проверки"
