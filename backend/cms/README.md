# Структура директорий Strapi CMS

## 📁 backend/cms/

### Назначение папок:

- **`src/`** — ваши кастомные типы контента, контроллеры, сервисы, политики
  - `api/` — кастомные API и модели
  - `components/` — повторяемые компоненты
  - `extensions/` — расширения стандартных функций

- **`plugins/`** — ваши кастомные плагины
  - Каждый плагин в отдельной подпапке
  - Пример: `plugins/my-custom-plugin/`

- **`config/`** — конфигурация Strapi
  - `admin.js` — настройки админки
  - `api.js` — настройки API
  - `database.js` — настройки БД (автоматически)
  - `middlewares.js` — мидлвары
  - `plugins.js` — включенные плагины

- **`public/uploads/`** — загруженные файлы (том Docker)

## 🚀 Как добавить плагин:

1. Создайте папку плагина: `backend/cms/plugins/my-plugin/`
2. Добавьте структуру плагина Strapi
3. Включите в `config/plugins.js`
4. Пересоберите: `docker compose up --build strapi`

## 📦 Установка npm-зависимостей:

Если плагину нужны дополнительные пакеты:
1. Добавьте их в `backend/cms/package.json`
2. Они установятся при сборке образа

## 🔧 Системные зависимости:

Если плагину нужны системные библиотеки (например, vips для обработки изображений):
- Раскомментируйте строку `RUN apk add ...` в `Dockerfile`
